# 经验沉淀规则

## 目的

每次事件（Bug、故障、决策、优化）都沉淀为团队可复用的知识资产。

---

## 触发条件

以下事件必须触发经验沉淀：

- P0/P1 Bug 修复
- 线上事故
- 重大架构决策
- 性能优化（QPS 提升 20% 以上）
- 用户反馈导致功能变更

---

## 沉淀流程

```
事件发生
  → 问题记录
  → 根因分析
  → 方案记录
  → 防复发措施
  → 写入经验库
  → 团队同步
```

---

## 经验库结构

```
docs/lessons/
├── index.md              # 经验索引
├── bugs/                 # Bug 类经验
│   ├── 2024-06-19-xxx.md
│   └── ...
├── architecture/         # 架构类经验
│   ├── sceneir-optimization.md
│   └── ...
└── performance/          # 性能类经验
    ├── render-performance.md
    └── ...
```

---

## 内容要求

### Bug 类经验

```
## 问题
## 根因
## 修复
## 如何提前发现
## 如何防止重复
```

### 架构类经验

```
## 背景
## 方案对比
## 选择理由
## 效果
## 注意事项
```

### 性能类经验

```
## 背景
## 性能瓶颈
## 优化方案
## 效果数据
## 通用建议
```

---

## 复查机制

- 每月回顾经验库
- 标记过时经验
- 补充遗漏经验
- 检查防复发措施是否落实

---

## 禁止

- 沉淀与代码注释重复的内容
- 沉淀临时且不再复现的问题
- 沉淀过于简单的内容（如"修了一个 null pointer"）

---

## 考核

- 每次 P0 Bug 必须有经验沉淀
- 经验沉淀需经过 Review
- 经验长期不更新需调查原因
