---

## 压缩触发条件

当以下情况之一出现时执行压缩：

1. 当前对话长度接近上下文窗口限制（约 80%）
2. 用户超过 5 轮没有引用早期对话内容
3. 会话中累积了大量工具调用结果
4. 模型主动检测到上下文正在膨胀

## 压缩策略

### 1. 结构化摘要

保留以下关键信息：
- 当前任务目标和进度
- 未完成的工作项
- 关键决策和理由
- 已修改的文件列表
- 最近的构建/部署状态
- 用户偏好（模型选择、风格要求等）

### 2. 丢弃

可以安全丢弃的信息：
- 已完成的工具调用输出
- 已修复的错误堆栈
- 已解决的讨论
- 中间状态的代码片段（最终版本已确认）

### 3. 格式

```
## 会话摘要

### 当前任务
{正在进行的任务}

### 进度
[X/Y] 已完成
- [x] 已完成事项
- [ ] 待完成事项

### 关键决策
- {决策1} → {理由}
- {决策2} → {理由}

### 已修改文件
- {文件路径} — {改动说明}

### 活跃环境
- 分支: {branch}
- 上次构建: {状态}
- 上次部署: {时间}

### 用户偏好
- 默认模型: Flash
- 风格要求: {如有}
```

## 注意事项

- 压缩后告知用户已进行压缩
- 不丢失任何未完成的工作上下文
- 保留所有用户明确要求的记忆点
