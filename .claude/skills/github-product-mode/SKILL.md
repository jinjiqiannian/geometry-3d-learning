# GitHub Product Mode Skill

This repository is a PRIVATE PRODUCT SYSTEM.

---

## Core Principle

GitHub is NOT documentation for developers.

It is a product presentation layer.

---

## README Rules

Must include:

- product definition
- what it does (user-facing)
- why it exists
- live demo link

Must NOT include:

- internal architecture deep detail
- prompt system explanation
- engineering complexity showcase

---

## Communication Style

- product-oriented
- user-focused
- outcome-driven

NOT technical storytelling.

---

## Goal

Make repository understandable in 10 seconds as a product.

---

## Enforcement

When modifying README or repo-facing docs:
1. Does it speak to users or developers? → Users
2. Does it explain internal architecture? → Remove
3. Does it show prompt/system details? → Remove
4. Can a non-technical person understand it in 10s? → Must
